# Mobitolya Support Routing

## Primary Message Types

Most Mobitolya traffic should be classified first as:

1. usage question
2. troubleshooting issue
3. configuration problem
4. workflow/data inconsistency
5. feature request

## Common Support Examples

- why does this not work
- how do I do this
- where is this setting
- why did this order/invoice/payment not move forward
- why is stock incorrect
- why did incoming WhatsApp content not parse correctly

## Response Style

Default response style should be:

- short
- practical
- low-jargon
- next-step oriented

## When To Escalate To Code Lookup

Look at repo code only when:

- the reported behavior may be a real product defect
- context is insufficient to explain the behavior
- there is a contradiction between expected and actual behavior
- the message references a concrete endpoint, screen, parser, worker, or flow
